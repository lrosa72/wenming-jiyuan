# S04 — 合规与一致性审查

---

## 技能名称
**合规与一致性审查**

## 技能编号
S04

## 技能类型
子技能 — 七步循环第④步

## 适用场景
- 正文生成后，必须通过本审查才能进入打磨
- 修改后再次审查

## 核心职责

> **检查规则合规、人物 OOC（Out of Character）、剧情矛盾。**
> 这是质量控制的第一个闸门，不通过不能进入下一步。

## 三大审查维度

```
合规与一致性审查
├── 维度一：平台规则合规
├── 维度二：人物 OOC 检测
└── 维度三：剧情矛盾检测
```

## 输入格式
```
- 章节正文（来自 S03 输出）
- 人物设定文档（所有人设卡）
- 世界观设定文档
- 前一章正文（用于连贯性检测）
- 合规规则清单（来自 技能配置/）
```

## 执行步骤

1. **规则加载**：加载最新的合规规则清单
2. **合规扫描**：逐类扫描敏感内容
3. **人物对照**：将本章每个人物的行为与人物设定卡逐一对照
4. **OOC 判断**：区分"合理的人物成长"和"真正的 OOC"
5. **设定冲突检测**：将本章内容与世界观设定逐条比对
6. **时间线校验**：检查时间推进是否合理
7. **前文矛盾检测**：与前一章对照，检查是否有直接矛盾
8. **能力体系校验**：检查战力/能力使用是否符合已确立的规则
9. **严重度评估**：对每个问题标注严重程度
10. **报告输出**：生成结构化审查报告

## OOC 判断标准

| 情况 | 判定 | 处理 |
|------|------|------|
| 人物行为与核心性格直接冲突 | 严重 OOC | 必须修改 |
| 人物说出不符合其说话风格的台词 | 中度 OOC | 建议修改 |
| 人物能力超过当前设定等级 | 严重 OOC | 必须修改 |
| 人物在压力下的反常行为（有铺垫） | 合理发展 | 无需修改 |
| 人物因剧情推进而成长变化（有过程） | 合理发展 | 无需修改 |
| 配角行为扁平化（工具人化） | 轻微偏差 | 建议关注 |

## 注意事项

- 合规审查是**硬性门槛**，不通过不能进入步骤⑤
- OOC 检测不是要阻止人物成长，而是确保变化**有动机、有过程**
- 剧情矛盾检测关注的是**直接矛盾**，不是"与读者预期不符"
- 审查报告中的每条问题都必须附带**具体修改建议**
- 连续 3 次审查不通过，触发主控的人工介入机制
- 不要过度审查——网文允许适当的夸张和戏剧化
