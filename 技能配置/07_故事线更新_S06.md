# S06 — 故事线与人物线更新

---

## 技能名称
**故事线与人物线更新**

## 技能编号
S06

## 技能类型
子技能 — 七步循环第⑥步

## 适用场景
- 每章打磨完成后自动调用
- 阶段性回顾时需要查看追踪数据

## 核心职责

> **自动更新世界观文档和伏笔追踪。**
> 本技能维护两套数据：故事线（伏笔+时间线+关系网）和人物线（弧光+状态+成长）。

## 输入格式
```
- 打磨后的章节正文（来自 S05 输出）
- 已有的伏笔追踪清单（来自 设定文档/故事线追踪/）
- 已有的人物弧光表（来自 设定文档/故事线追踪/）
- 人物设定文档
- 世界观设定文档
- 名词术语表
```

## 执行步骤

1. **加载追踪数据**：读取所有追踪文件的最新版本
2. **新章解析**：从打磨后正文中提取伏笔操作、人物变化、时间信息、关系变化
3. **伏笔更新**：标注已回收、添加新埋伏、检查超期未回收
4. **人物弧光更新**：评估每个人物的状态变化，推进弧光进度
5. **时间线推进**：更新时间节点，校验时间逻辑
6. **关系网络更新**：记录新增和变化的人物关系
7. **设定变更检测**：识别新设定和设定修改，触发 S07
8. **异常检测**：标记伏笔遗漏、时间矛盾、人物突变、关系跳跃
9. **追踪文件写入**：将更新写入对应的追踪数据文件
10. **报告输出**：生成追踪更新报告

## 追踪数据文件结构

```
设定文档/故事线追踪/
├── 伏笔清单.md       # 所有伏笔的完整生命周期
├── 人物弧光.md       # 每个人物的弧光推进状态
├── 时间线.md         # 故事时间线
├── 关系网络.md       # 人物关系图谱
└── 更新日志.md       # 每次更新的简要记录
```

## 注意事项

- **伏笔生命周期管理**：埋设→强化→回收，每个阶段都要记录时间戳
- **超期预警**：超过 10 章未回收的伏笔 ⚠️，超过 20 章 🚨
- **人物弧光不跳跃**：人物的变化必须逐章积累，不能突然转变
- **时间线精度**：保持天级精度即可
- **关系网络只记录重要关系**：路人互动不纳入
- 追踪数据是 S01（资料整理）的重要输入，形成信息闭环
