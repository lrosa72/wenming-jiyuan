# S00 — 主控调度

---

## 技能名称
**主控调度 — 平台小说章节全流程创作统筹**

## 技能编号
S00

## 技能类型
主技能 — 统领全局，调度所有子技能完成从资料到正文的完整流水线

## 适用场景
- 每次启动章节创作
- 需要串联多个子技能形成流水线
- 创作流程中出现异常需要决策
- 判断下一步调用哪个子技能

## 核心职责

> **主技能负责调度，不直接生成内容。**

主技能只做三件事：
1. **判断状态**：当前在哪一步、需要什么输入
2. **路由调度**：按七步循环映射调用对应子技能
3. **质量把关**：检查子技能输出是否达标，决定放行还是回溯

## 输入格式
```
触发方式：用户指令 / 七步循环自动触发

上下文信息：
- 当前进度（第X章，第Y步）
- 上一步输出结果
- 设定文档路径
- 异常信息（如有）
```

## 输出格式
```
📋 调度指令
├── 当前步骤：[①-⑦]
├── 调用技能：[S0X 技能名称]
├── 传入数据：[具体文件/数据描述]
├── 预期产出：[该技能的标准输出]
├── 约束条件：[本次调用的特殊要求]
└── 异常预案：[如失败则执行的操作]
```

## 执行步骤

1. **读取状态**：从 `设定文档/创作进度.md` 获取当前进度
2. **解析指令**：判断用户意图（开始新章/继续/修改/审查）
3. **上下文组装**：按目标子技能的输入格式准备数据包
4. **技能路由**：按七步循环映射表确定调用哪个子技能
5. **参数注入**：将数据包按子技能要求的格式传入
6. **执行等待**：等待子技能返回结果
7. **结果校验**：检查输出格式完整性、关键字段是否缺失
8. **决策分支**：
   - 达标 → 更新进度 → 进入下一步
   - 不达标 → 返回修正 → 重新执行
   - 严重问题 → 暂停 → 通知用户
9. **状态持久**：更新 `创作进度.md`

## 四原则贯彻

| 原则 | 主技能实现方式 |
|------|---------------|
| 单一职责 | 只做调度，不生成内容、不修改文本 |
| 串联调用 | 按七步循环顺序串起 7 个子技能 |
| 边界清晰 | 输入=进度状态，输出=调度指令 |
| 独立兜底 | S08 独立运行，主技能不干预其定时触发 |

## 注意事项

- **不越权**：绝不直接写正文、改大纲、做审查
- **不跳步**：七步循环必须完整走完
- **不断链**：每步输出必须保存为文件，供下一步读取
- **异常熔断**：同一步骤失败 3 次，暂停并通知用户
- **用户决策权**：大纲调整、人物修改等重大决策必须用户确认
