# S08 — 每日存稿备份与合规扫描

---

## 技能名称
**每日存稿备份与合规扫描**

## 技能编号
S08

## 技能类型
自动化技能 — 独立于主流程，定时自动执行

## 适用场景
- 每日定时自动触发（如凌晨 3:00）
- 全项目健康检查
- 灾难恢复

## 核心职责

> **自动化技能独立于主流程，做兜底保障。**
> 每日自动备份全项目 + 扫描敏感词和违规风险。不参与七步循环，独立运行。

## 与 S07 的区别

| 维度 | S07（本章备份） | S08（每日兜底） |
|------|----------------|-----------------|
| 触发方式 | 七步循环第⑦步自动触发 | 定时触发（每日） |
| 备份范围 | 本章相关文件 | **全项目文件** |
| 备份粒度 | 章节级 | 日级全量 |
| 合规扫描 | 不做（S04 已做） | **全项目敏感词扫描** |
| 目的 | 章节级版本管理 | **灾难恢复 + 合规兜底** |
| 与主流程关系 | 主流程内 | **主流程外，独立运行** |

## 输入格式
```
触发方式：定时触发（每日固定时间）/ 手动触发

扫描范围：
├── 全项目 .md 文件
├── 正文存稿/ 全部章节
├── 大纲细纲/ 全部文件
└── 设定文档/ 全部文件
```

## 执行步骤

1. **定时唤醒**：每日固定时间自动触发（建议凌晨低负载时段）
2. **全项目扫描**：遍历所有 .md 文件，建立文件清单
3. **变更检测**：与上一次 S08 备份对比，识别变更文件
4. **全量备份**：将所有项目文件打包备份
5. **敏感词扫描**：
   - 加载敏感词库
   - 逐文件扫描所有正文存稿
   - 分类标记命中项
6. **项目健康检查**：
   - 检查五个核心目录是否存在
   - 检查进度文件与实际文件是否一致
   - 检查追踪数据是否完整
   - 检查备份链是否有断裂
7. **待办事项汇总**：从 S06 追踪数据中提取待处理事项
8. **报告输出**：生成每日兜底报告
9. **通知推送**：如有高危问题，主动通知用户

## 敏感词库维护

```
技能配置/敏感词库/
├── 高危词库.md    # 必须立即处理的敏感词
├── 中危词库.md    # 需关注和评估的敏感词
└── 低危词库.md    # 需注意的边界词汇
```

> 敏感词库需要根据平台规则持续更新。初始版本由用户提供或根据目标平台规则整理。

## 注意事项

- **独立运行**：S08 不在七步循环内，即使创作流程暂停也会按时执行
- **不修改内容**：只扫描和报告，不自动修改任何文件
- **静默运行**：正常情况下后台执行，只在发现问题时通知用户
- **兜底不是替代**：S08 的合规扫描是补充性的，不能替代 S04 的逐章审查
- **备份冗余**：S07（章节级）+ S08（日级全量）= 双重保障
- **磁盘空间管理**：日备份文件较大，需定期清理超过 90 天的日备份
