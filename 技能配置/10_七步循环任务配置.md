# 七步循环任务配置

> 这是网文创作的核心引擎——单章创作的工业流水线。
> 每一章都严格按照这 7 个步骤执行，每步有明确的输入、输出和技能调用。
> **在启动流水线之前，必须先完成"前置环节：创作讨论"。**

---

## 前置环节：创作讨论 ⭐

> **在七步循环启动之前，用户与 WorkBuddy 必须先讨论本章的大致方向。**
> 这个环节的目的是引入人的创作意图，避免纯 AI 生成。

### 讨论流程

```
用户提出本章大致想法
        ↓
WorkBuddy 读取资料包（前情提要 + 人物状态 + 伏笔清单）
        ↓
WorkBuddy 搜索同类网文作为参考（可选）
        ↓
双方讨论：
├── 本章核心冲突方向
├── 想要的情节发展和爽点
├── 需要回收的伏笔
├── 参考作品的写法借鉴
└── 特别注意事项（如某角色需重点刻画）
        ↓
达成共识 → 启动七步循环
```

### 讨论输入

- 用户对本章的想法（一句话或一段话即可）
- S01 预整理的资料包（前情提要 + 人物状态 + 伏笔清单）
- 可选：同类网文搜索结果

### 讨论输出

```
🎯 第X章 创作共识

├── 核心方向：[本章要完成的核心任务]
├── 关键情节：[用户指定的必须出现的情节]
├── 情感走向：[本章的整体情绪基调]
├── 参考借鉴：[同类作品的处理方式]
├── 特别要求：[用户的特殊指示]
└── 避坑提醒：[需要注意避免的问题]
```

### 讨论原则

- **用户主导方向，AI 提供参考**：创作意图来自用户，AI 负责提供资料、参考和可行性分析
- **搜索同类网文是可选步骤**：如果用户已有清晰想法可以跳过，如果不确定方向则搜索参考
- **讨论不替代细纲**：讨论确定的是方向，具体的场景拆分和节奏设计仍由 S02 完成
- **共识驱动流水线**：讨论结论作为约束条件，贯穿整个七步循环

## 四原则

> 9 个技能的设计和运行遵循四个核心原则：

| 原则 | 说明 | 贯彻方式 |
|------|------|----------|
| **单一职责** | 每个技能只做一件事，输入输出边界清晰 | 每个技能文件明确定义唯一的职责和边界 |
| **串联调用** | 技能之间可以串联调用，形成流水线 | 七步循环：上一步输出 = 下一步输入 |
| **主技能不生成内容** | 主技能负责调度，不直接生成创作内容 | S00 只做路由、校验、状态管理 |
| **自动化独立兜底** | 自动化技能独立于主流程，做兜底保障 | S08 定时运行，不参与七步循环 |

## 步骤映射表

| 步骤 | 名称 | 调用技能 | 输入来源 | 输出去向 |
|------|------|----------|----------|----------|
| 前置 | 创作讨论 | S00（主控）+ WebSearch | 用户意图 + 资料包 + 同类网文 | 七步循环 |
| ① | 资料整理 | **S01**（资料整理） | 前章正文 + S06追踪数据 + 设定文档 | 步骤② |
| ② | 剧情推演 | **S02**（剧情推演） | 步骤①资料包 + 大纲 + 创作共识 | 步骤③ |
| ③ | 正文生成 | **S03**（平台正文生成） | 步骤②细纲 + 设定 + 前文 + 风格指南 | 步骤④ |
| ④ | 合规审查 | **S04**（合规与一致性） | 步骤③正文 + 人设卡 + 设定文档 | 步骤⑤（通过）/ 步骤③（不通过） |
| ⑤ | 内容打磨 | **S05**（内容优化打磨） | 步骤④通过的正文 + 风格指南 | 步骤⑥ |

## 每一步详细说明

### 前置：创作讨论

**目标**：在启动流水线之前，用户与 WorkBuddy 讨论本章大致方向，避免纯 AI 生成。

**调用技能**：
- **S00 主控调度**：协调讨论流程，加载资料包
- **WebSearch**（可选）：搜索同类网文作为参考

**具体操作**：
1. 用户提出本章大致想法
2. S00 加载预整理的资料包（前情提要 + 人物状态 + 伏笔清单）
3. 如需要，搜索同类网文的情节处理方式作为参考
4. 双方讨论核心冲突、关键情节、情感走向
5. 用户补充想要的情节发展
6. 达成共识，输出创作共识文档

**输出**：`大纲细纲/第X章_创作共识.md`

**检查点**：
- [ ] 用户已确认本章核心方向
- [ ] 关键情节已明确
- [ ] 参考借鉴（如有）已记录
- [ ] 特别要求和避坑提醒已记录

---

### 步骤①：资料整理与前文梳理

**目标**：AI 自动读取前文状态、人物变化、未回收伏笔。只整理不创作。

**调用技能**：
- **S01 资料整理与前文梳理**：收集归纳所有前文信息

**具体操作**：
1. S00 主控调度将进度状态和创作共识传给 S01
2. S01 读取前一章正文，提取核心事件和结尾钩子
3. 从 S06 追踪数据中提取所有人物最新状态和变化
4. 整理待回收伏笔，按优先级排列
5. 对照大纲确认本章定位和使命
6. 匹配本章可能涉及的设定条目

**输出**：`大纲细纲/第X章_资料包.md`（前情提要 + 人物状态与变化 + 伏笔清单 + 大纲定位 + 设定提醒）

**检查点**：
- [ ] 所有出场人物的状态已更新
- [ ] 人物变化（能力/关系/心态）已标注
- [ ] 待回收伏笔已按优先级排列
- [ ] 前一章结尾钩子已明确
- [ ] 本章大纲定位已确认

---

### 步骤②：单章剧情推演

**目标**：基于大纲和细纲，推演本章核心冲突与转折。

**调用技能**：
- **S02 单章剧情推演**：将大纲节点展开为本章细纲

**具体操作**：
1. S02 完整阅读 S01 输出的资料包和创作共识
2. 在大纲中定位本章节点
3. 提炼本章核心冲突和转折
4. 拆分 2-4 个具体场景
5. 设计情感/紧张度曲线
6. 规划伏笔操作（回收 + 新建）
7. 设计开篇钩子和结尾钩子
8. 校验与前后章的衔接

**输出**：`大纲细纲/第X章_细纲.md`（场景清单 + 节奏曲线 + 伏笔计划 + 钩子设计）

**检查点**：
- [ ] 核心冲突明确，与创作共识一致
- [ ] 场景数量 2-4 个
- [ ] 每个场景有信息增量
- [ ] 开篇和结尾钩子已设计
- [ ] 本章有不可逆的剧情转折
- [ ] 新埋伏笔 ≤ 2 个

---

### 步骤③：平台标准正文生成

**目标**：生成约 2000 字初稿，内置去 AI 味处理。

**调用技能**：
- **S03 平台标准正文生成**：执行正文创作 + 去 AI 味

**具体操作**：
1. S03 加载细纲、风格指南、风格样本、人物设定
2. 阅读风格样本，进入写作状态
3. 按细纲场景顺序逐个创作（先画面，后文字）
4. 自然嵌入伏笔回收和新埋
5. 实现开篇和结尾钩子
6. 执行去 AI 味自检（8 项检查）
7. 字数校验（1800-2200 字）
8. 生成章节元数据

**输出**：`正文存稿/第X章-[标题].md`（正文 + 元数据 + 去AI味自评）

**检查点**：
- [ ] 未偏离细纲核心冲突
- [ ] 人物言行符合设定
- [ ] 去 AI 味自检通过（8/8）
- [ ] 字数 1800-2200
- [ ] 章节标题已确定
- [ ] 开篇钩子和结尾钩子到位

---

### 步骤④：合规与一致性审查

**目标**：检查 OOC、剧情矛盾、平台规则合规。

**调用技能**：
- **S04 合规与一致性审查**：三维审查（合规 + OOC + 矛盾）

**具体操作**：
1. S04 加载合规规则清单、人物设定卡、世界观设定
2. 维度一：平台规则合规扫描
3. 维度二：人物 OOC 检测（行为 vs 人设卡对照）
4. 维度三：剧情矛盾检测（设定冲突/时间矛盾/前文矛盾/逻辑漏洞/能力崩坏）
5. 严重度评估和修改建议生成

**输出**：审查报告（三维结果 + 问题清单 + 修改建议 + 处理意见）

**决策分支**：
- ✅ **通过** → 进入步骤⑤
- ⚠️ **有条件通过** → 修改后复查 → 进入步骤⑤
- ❌ **不通过** → 返回步骤③重写问题段落 → 重新审查（最多 3 次）

**检查点**：
- [ ] 三维审查全部完成
- [ ] 高严重度合规问题清零
- [ ] 严重 OOC 问题清零
- [ ] 直接剧情矛盾清零
- [ ] 每条问题附具体修改建议

---

### 步骤⑤：内容优化打磨

**目标**：润色对话、增强画面感、调整章末钩子。

**调用技能**：
- **S05 内容优化打磨**：三大方向打磨（对话/画面/节奏+钩子）

**具体操作**：
1. S05 重新阅读风格样本，建立打磨标准
2. 以读者视角通读全文，标注感受卡顿处
3. 方向一：对话润色（角色辨识度 + 潜台词 + 去标签化）
4. 方向二：画面感增强（五感描写 + 具象化 + 动作链条）
5. 方向三：节奏调整（段落长短 + 句式变化 + 过渡优化）
6. **章末钩子专项检查**：确保结尾钩子有足够的悬念/期待驱动力
7. 基础纠错（错别字/标点/语病）
8. 冗余删除
9. 变更标注

**输出**：`正文存稿/第X章-[标题]_打磨稿.md`（打磨后正文 + 打磨报告）

**决策分支**：
- 打磨满意 → 进入步骤⑥
- 发现重大问题 → 返回步骤③重写
- 打磨过度 → 回退部分修改

**检查点**：
- [ ] 字数变化 ≤ ±10%
- [ ] 所有修改已标注原因
- [ ] 对话有角色辨识度
- [ ] 关键场景有画面感
- [ ] 节奏有呼吸感（有快有慢）
- [ ] **章末钩子有驱动力**（读者会想翻下一章）

---

### 步骤⑥：故事线与人物线更新

**目标**：自动更新人物关系表、伏笔追踪表。

**调用技能**：
- **S06 故事线与人物线更新**：更新伏笔/人物弧光/时间线/关系网络

**具体操作**：
1. S06 加载所有追踪数据文件
2. 从打磨稿解析伏笔操作、人物变化、时间信息、关系变化
3. 更新伏笔清单（回收/新增/强化/超期预警）
4. 更新人物弧光状态（每个出场人物）
5. 更新人物关系表
6. 推进时间线
7. 检测设定变更，同步知识库
8. 异常检测和标记

**输出**：追踪更新报告 + 追踪数据文件更新（伏笔清单/人物关系表/人物弧光/时间线）

**检查点**：
- [ ] 伏笔状态全部更新
- [ ] 人物关系表已更新
- [ ] 人物弧光已推进
- [ ] 时间线无矛盾
- [ ] 超期伏笔已预警

---

### 步骤⑦：自动备份

**目标**：每章存档，保留完整版本历史。工业流水线的最后一道工序。

**调用技能**：
- **S07 自动版本备份**：章节级版本快照

**具体操作**：
1. S07 检测本章所有变更文件
2. 生成版本号 V[卷].[章].0
3. 复制文件到版本目录
4. 完整性校验
5. 版本链更新
6. 过期版本清理

**输出**：备份报告 + `版本备份/V[卷].[章].0/`

**检查点**：
- [ ] 本章所有文件已备份
- [ ] 版本号正确
- [ ] 完整性校验通过
- [ ] 备份日志已更新

## 异常处理流程

```
异常发生
    │
    ├── 前置讨论无法达成共识 → 继续讨论或用户给出明确指示
    │
    ├── 步骤①-②异常 → 检查设定/大纲完整性 → 补充后再执行
    │
    ├── 步骤③异常 → 检查细纲是否合理 → 必要时返回步骤②
    │
    ├── 步骤④不通过 → 修改正文 → 重审（最多3次）
    │       └── 3次仍不通过 → 返回步骤③重写 → 再审查
    │
    ├── 步骤⑤发现重大问题 → 返回步骤③ → 重走后续步骤
    │
    └── 步骤⑥-⑦异常 → 记录错误日志 → 人工介入
```

## 阶段审查触发机制

七步循环内嵌了阶段审查的触发逻辑。每完成 5 章，自动插入阶段审查步骤。

### 触发规则

```
IF 当前章节号 % 5 == 0:
    七步循环完成后 → 触发「阶段审查」
    ├── 切换模型到 DeepSeek
    ├── 审查范围：最近 5 章（第 N-4 章 ~ 第 N 章）
    ├── 审查维度：逻辑/连贯性/文笔/伏笔/人物弧光
    └── 输出：阶段审查报告 + 修改清单
```

### 阶段审查在流程中的位置

```
... → ⑦自动备份 → [5章?] → 是 → 🔍 阶段审查（DeepSeek）
                      ↓否
                   下一章 [前置] 创作讨论
```

### 阶段审查后的处理

```
阶段审查完成
    ├── 无问题 → 继续下一章创作
    ├── 轻微问题 → 记录，后续章节注意
    ├── 中等问题 → 排入修改队列，不阻塞创作
    └── 严重问题 → 暂停创作，立即修改 → 修改后继续
```
